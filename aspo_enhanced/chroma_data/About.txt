# AI Chat — ChatGPT-style Flask App

A production-quality AI chat application powered by **Groq API** (GPT-OSS 120B), built with Flask, SQLite, and vanilla JS. Features persistent multi-chat history, streaming responses, markdown rendering, and a dark ChatGPT-inspired UI.

---

## Features

- **Streaming responses** via Server-Sent Events (SSE)
- **Persistent chat history** in SQLite — survives restarts and refreshes
- **Multiple independent chats** with auto-generated titles
- **Markdown + syntax highlighting** for code blocks
- **Copy message / copy code** buttons
- **Mobile responsive** with collapsible sidebar
- **ChatGPT-style dark UI** — no white anywhere

## Setup & Installation

### 1. Clone / copy the project

```bash
cd project
```

### 2. Create and activate a virtual environment (recommended)

```bash
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set your Groq API key

**Option A — environment variable (recommended):**

```bash
export GROQ_API_KEY="gsk_your_key_here"   # Linux / macOS
set GROQ_API_KEY=gsk_your_key_here        # Windows CMD
$env:GROQ_API_KEY="gsk_your_key_here"     # Windows PowerShell
```

**Option B — edit `app.py` directly (dev only):**

```python
GROQ_API_KEY = "gsk_your_key_here"
```

Get a free API key at: https://console.groq.com

### 5. Run the app

```bash
python app.py
```

Open your browser at **http://localhost:5000**

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET  | `/api/chats` | List all chats |
| POST | `/api/chats` | Create new chat |
| DELETE | `/api/chats/<chat_id>` | Delete a chat |
| GET  | `/api/chats/<chat_id>/messages` | Get messages for a chat |
| POST | `/api/chats/<chat_id>/messages` | Send a message (streams response) |

---

## Production Deployment

For production, use **Gunicorn** with an async worker:

```bash
pip install gunicorn gevent
gunicorn -w 1 -k gevent --worker-connections 1000 -b 0.0.0.0:5000 app:app
```

> Use `-w 1` with streaming/SSE to avoid connection issues across workers.

---

## Notes

- The database (`database.db`) is created automatically on first run.
- All chat messages persist permanently — refreshing or restarting the server won't delete them.
- The model used is `openai/gpt-oss-120b` via Groq's OpenAI-compatible endpoint.
